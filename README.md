# ECD Decision Support Layer — MVP

AI-based developmental risk screening for children aged 0–72 months.
Built for the ICDS / WD&CW Innovation Challenge.

## Quick Start (3 commands)

```bash
pip install -r requirements.txt
python seed_demo_db.py
streamlit run app.py
```

The app opens at `http://localhost:8501`

---

## Demo Flow

### Key Demo Children (pre-seeded)
| Child ID | Trajectory | Demo Purpose |
|---|---|---|
| ECD-0001 | Low → Low (stable) | Shows normal screening output |
| ECD-0002 | High → Medium (improving) | Shows positive trajectory detection |
| **ECD-0003** | **Medium → Critical** | **KEY DEMO: system catches deterioration** |

### Recommended Demo Path (10 minutes)
1. **Start as AWW** → Select a child → Complete consent → Run screening → Show risk result + SHAP
2. **Trigger referral gate**: Enter low indicators for ECD-0003 child → Critical tier forces action
3. **Switch to Supervisor** → Block Dashboard → Cohort metrics → Trajectory chart for ECD-0003
4. **Show Audit Trail** → Every action logged
5. **About page** → Model card metrics, DPDP statement, government ownership

### Offline Simulation
Toggle "📴 Offline Mode" in sidebar → App uses rule-based provisional tier (no model required) → UI shows amber warning banner.

## File Structure

```
ECD/
├── app.py                      ← Main Streamlit application
├── schema.py                   ← SQLite schema + shared constants
├── risk_engine.py              ← XGBoost model, SHAP, calibration
├── trajectory_engine.py        ← Developmental coefficient, transitions
├── cohort_aggregator.py        ← Block/district metrics
├── synthetic_data_generator.py ← Synthetic dataset (WHO/DASII norms)
├── decision_trace.py           ← Governance audit log
├── seed_demo_db.py             ← Database seeding script
├── requirements.txt            ← Pinned dependencies
├── demo.db                     ← Generated by seed_demo_db.py
└── model/
    └── risk_model.joblib       ← Generated by seed_demo_db.py
```
